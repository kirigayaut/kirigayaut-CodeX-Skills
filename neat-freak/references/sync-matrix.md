# 变更影响矩阵

仅在 durable 变化涉及多个知识层，或影响范围不明确时读取。矩阵给出候选位置，不要求每次创建或修改所有文件；以项目现有文档结构和受众为准。

## 信息路由

| 信息 | 主位置 | 不应放在 |
|---|---|---|
| 必须遵守的工程约束、验证命令、红线 | `AGENTS.md` / `CLAUDE.md` | changelog、Memory 唯一副本 |
| 项目目的、安装和最短启动路径 | `README.md` | 全局 Agent 规则 |
| 架构、数据流、接口、运维和接入说明 | 对应 `docs/` | 规则文件全文复制 |
| 已确认的长期设计决定及理由 | ADR / `docs/decisions.md` | 临时任务摘要 |
| 当前里程碑、待办和下一步 | progress 文档或任务系统 | `AGENTS.md` |
| 发布、修复和退役历史 | changelog 或 git history | 活跃规则段落 |
| 临时推测、调试日志、一次性问答 | 当前任务 | durable 文档 |

## 变化类型

| 变化 | 检查候选 |
|---|---|
| API、路由或协议 | 接口参考、集成指南、架构；仅当 Agent 必须遵守相关边界时更新规则文件 |
| 环境变量或配置键 | 配置示例、安装说明、runbook；安全红线再进入规则文件 |
| 数据模型或持久化格式 | schema、架构、迁移或兼容性说明 |
| 构建、测试、lint 命令 | `AGENTS.md`、开发指南、CI 配置 |
| 用户流程或 CLI 用法 | README、用户指南、示例 |
| 部署或基础设施 | runbook、部署说明、配置模板 |
| 权限或安全边界 | 规则文件、security 文档；可机械强制的反复违规考虑 hook/rule |
| 长期设计决定 | ADR / decisions 文档，并更新受影响的架构说明 |
| 当前阶段状态 | progress / issue tracker，不写入规则文件 |
| 文件、功能或接口退役 | 搜索并清除活跃说明中的引用，保留历史记录的准确语境 |
| 跨项目共享协议 | 上游权威说明和有证据的直接消费者文档 |

## 删除与改名

1. 从 `git diff`、提交或用户说明取得真实旧名称和新名称。
2. 搜索代码、配置、规则和文档中的活跃引用。
3. 删除或更新会误导当前使用者的引用。
4. 保留 changelog、ADR 和历史案例里的历史事实，必要时标注版本或日期。
5. 不直接修改 Codex 生成式 Memories 来追求搜索结果清零。

## 冲突处置

| 情况 | 动作 |
|---|---|
| 代码、测试和已确认决定一致，文档明显过期 | 更新文档 |
| 文档描述目标设计，但实现尚未完成 | 保留目标设计并明确状态，不假装已落地 |
| 两份规则文件有明确覆盖顺序 | 按覆盖顺序处理，并消除无意义重复 |
| 两份规则无覆盖顺序且内容冲突 | 不自动合并，列出差异等待确认 |
| 修复只涉及安全、可逆的文档纠错 | 直接修复并报告 |
| 修复涉及删除、重命名、部署或外部系统 | 先取得授权 |

## 最小修改原则

- 优先更新已有权威条目，不新建平行文档。
- 同一事实只指定一个主位置，其他位置使用必要的短引用。
- 只有受众确实不同，才在多个层分别表达。
- 单点进度更新不升级为 neat-freak 全量审计。
- 自动压缩或新开任务本身不产生文档修改义务。
