# Codex 知识边界

仅在任务涉及 Memories、规则加载链或 Skill 发现位置时读取本文件。以当前会话暴露的能力和最新官方文档为准，不凭目录名推断平台行为。

## 当前任务上下文

- 当前任务保留自己的对话记录和工作目录。
- 自动压缩或支持界面中的 `/compact` 用于精简当前任务的可见对话，不负责更新项目文档或长期规则。
- 新任务应重新读取适用的项目规则和文档；不要假设压缩摘要或旧任务包含全部现行事实。

## 项目指导

- Codex 从全局和项目目录加载适用的 `AGENTS.override.md`、`AGENTS.md` 以及明确配置的 fallback 文件。
- 必须每次生效的项目指导放在最接近适用范围的 `AGENTS.md`。
- 团队和项目事实放在检入版本控制的 `README.md`、`docs/`、配置、代码和测试中。
- 保持 `AGENTS.md` 简短；把详细的架构、运行手册和专题流程放在对应文档中并使用短链接。

## Memories

- Memories 是可选的跨任务召回层，不是必须执行规则的唯一来源。
- Codex 可能跳过活跃或短任务，并在后台延迟生成 Memory。
- `$CODEX_HOME/memories/` 是生成式状态；可以核验，但不要把手工编辑当作主要控制方式。
- 在支持的界面中使用 `/memories` 或产品设置控制当前任务是否使用 Memories、是否参与后续生成。
- 必须立即且可靠地保留的信息应进入正确的项目文件，而不是等待 Memory 生成。

## Skills

- Skills 打包可复用流程和领域知识，不承担某个项目现行事实的唯一存储责任。
- Codex 先加载 Skill 的名称和描述，触发后才加载 `SKILL.md`，再按需读取 references 或运行 scripts。
- 以当前会话暴露的 Skill 清单和路径为准；不要假设所有 Codex 表面使用同一安装目录。
- 修改 Skill 后校验 `SKILL.md` 与 `agents/openai.yaml`；未自动刷新时重启 Codex。

## 无法确认时

无法确认规则加载、Memory 写入或 Skill 发现行为时，保持相关状态只读，维护项目文件，并在结果中说明边界。
