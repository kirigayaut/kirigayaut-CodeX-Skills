# 平台知识边界

仅在任务涉及 Agent Memories、跨平台规则文件或 Skill 安装位置时读取本文件。先探测实际平台和可用能力，不要仅凭目录名推断。

## OpenAI Codex 5.6

### 权威项目知识

- 全局个人规则：`$CODEX_HOME/AGENTS.md`，默认根目录通常为 `~/.codex`。
- 项目规则：从项目根到当前目录生效的 `AGENTS.override.md`、`AGENTS.md` 及配置的 fallback 文件。
- 项目事实：检入版本控制的 `README.md`、`docs/`、配置、代码和测试。

### Memories

- 本地 Memories 位于 Codex home 下，由 Codex 在后台从合格任务生成。
- Memories 可能延迟更新，也可能跳过活跃或短任务，因此不能承担“必须马上落盘”的项目规则。
- 把 Memories 视为生成式辅助状态。可以在排障或核验时读取，但不要把手工编辑生成文件作为主要控制方式。
- 必须始终生效的规则写入 `AGENTS.md`；团队和项目事实写入检入版本控制的文档。
- 只有用户明确要求持久化 Memory，且当前运行环境提供受支持更新机制时才执行写入。

### Skills

- 以当前会话暴露的 Skill 目录和技能清单为准。
- 不要假设所有 Codex 表面、版本和托管环境使用同一安装路径。
- 修改 Skill 后校验 `SKILL.md` 与 `agents/openai.yaml`，必要时重启或新开任务刷新发现结果。

## Claude Code

- 常见项目规则文件为 `CLAUDE.md`，记忆和 Skill 位置取决于当前 Claude Code 版本和配置。
- Claude Code 的索引尺寸、加载截断和记忆文件生命周期规则只适用于已核实的 Claude Code 环境，不得套用到 Codex。
- 仅在用户授权且平台文档明确支持时修改记忆文件。

## OpenCode 与 OpenClaw

- 先读取当前平台的项目配置和 Skill 发现规则。
- 不要因为兼容读取其他平台目录，就假设其 Memory 写入语义也兼容。
- 没有受支持的独立 Memory 写入面时，只维护项目规则和项目文档。

## 跨平台项目

按以下顺序处理：

1. 确认每个平台实际读取哪个规则文件。
2. 确认项目已有的权威文件和覆盖关系。
3. 共享内容尽量落在平台中立的 `README.md` 和 `docs/`。
4. 规则文件可以互相引用、生成或使用软链，但不要强制一种全局方案。
5. 两份独立规则已经分叉时，不自动合并；列出差异并确认权威来源。

无法确认平台行为时，跳过 Memory 修改并在结果中说明边界。
